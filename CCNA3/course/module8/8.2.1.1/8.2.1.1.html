Topologie de réseau OSPF

Introduit en 1991, OSPFv2 est un protocole de routage à état de liens pour IPv4. Le protocole OSPF a été conçu comme alternative à un autre protocole de routage IPv4, le protocole RIP.

La figure montre la topologie utilisée pour configurer le protocole OSPFv2 dans cette section. Les types des interfaces série et les bandes passantes associées ne correspondent pas nécessairement aux types de connexions les plus courants figurant sur les réseaux actuels. Les bandes passantes des liens série utilisés dans cette topologie ont été choisies pour expliquer le calcul des métriques du protocole de routage et le processus de sélection du meilleur chemin.

Les routeurs de la topologie disposent d'une configuration initiale, qui inclut les adresses d'interface. Aucun routage statique ou dynamique n'est actuellement configuré sur l'un des routeurs. Toutes les interfaces des routeurs R1, R2 et R3 (sauf le bouclage sur R2) se trouvent dans la zone fédératrice OSPF. Le routeur du fournisseur d'accès à Internet est utilisé comme passerelle du domaine de routage pour accéder à Internet.

Remarque : dans cette topologie, l'interface de bouclage est utilisée pour simuler le lien de réseau étendu donnant accès à Internet.

La figure illustre trois routeurs configurés sous forme de triangle. Chaque routeur est connecté aux deux autres routeurs à l'aide de câbles série WAN. Chaque routeur dispose d'un LAN local configuré sur chaque interface Gigabit Ethernet. L'un des trois routeurs dispose d'une interface de bouclage configurée pour simuler une connexion à Internet. Les trois routeurs se trouvent dans la zone OSPF portant le numéro 0. Sept réseaux constituent cette topologie, laquelle est désignée en tant que topologie de référence pour le protocole OSPF dans les pages qui suivent.
