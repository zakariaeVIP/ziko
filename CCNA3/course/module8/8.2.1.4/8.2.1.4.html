Configuration d'un ID de routeur OSPF

Utilisez la commande router-id rid en mode de configuration de routeur pour attribuer manuellement une valeur 32 bits exprimée sous forme d'adresse IPv4 à un routeur. Un routeur OSPF s'identifie auprès des autres routeurs au moyen de cet ID de routeur.

Comme illustré dans la Figure 1, R1 est configuré avec l'ID de routeur 1.1.1.1, R2 avec 2.2.2.2 et R3 avec 3.3.3.3.

Dans la Figure 2, l'ID de routeur 1.1.1.1 est attribué à R1. Exécutez la commande show ip protocols pour vérifier l'ID de routeur.

Remarque : R1 n'avait jamais été configuré avec un ID de routeur OSPF. Si c'était le cas, l'ID du routeur devrait être modifié.

Si l'ID de routeur est identique sur deux routeurs voisins, le routeur affiche un message d'erreur similaire à celui-ci :

%OSPF-4-DUP_RTRID1: Detected router with duplicate router ID.

Pour corriger ce problème, configurez tous les routeurs de manière à ce qu'ils aient un ID OSPF unique.

Utilisez le contrôleur de syntaxe dans la Figure 3 pour attribuer un ID de routeur à R2 et R3.

La Figure 1 illustre la topologie de référence pour le protocole OSPF, ainsi que les valeurs des ID de routeur. L'ID configuré pour le routeur R1 équivaut à 1.1.1.1. L'ID configuré pour le routeur R2 équivaut à 2.2.2.2. L'ID configuré pour le routeur R3 équivaut à 3.3.3.3. La Figure 2 affiche les commandes utilisées pour attribuer un ID de routeur. La figure commence en mode de configuration globale. La première commande est : router OSPF. 10. Cette commande configure OSPF sur le routeur et attribue la valeur 10 au processus. La commande suivante est : router, trait d'union, ID, un, point, un, point, un, point, un. Cette commande définit l'ID de routeur. La figure présente ensuite le résultat de la commande, show ip protocols. La ligne des résultats affichés contenant l'ID de routeur est surlignée. La Figure 3 illustre un élément multimédia interactif qui permet à l'étudiant d'attribuer un ID à un routeur.
