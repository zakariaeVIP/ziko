Configuration des interfaces passives

Utilisez la commande du mode de configuration du routeur passive-interface pour empêcher la transmission de messages de routage via une interface de routeur, mais permettre que le réseau soit annoncé aux autres routeurs, comme illustré dans la Figure 1. En particulier, la commande interrompt l'envoi de messages de routage à partir de l'interface spécifiée. Toutefois, le réseau auquel appartient l'interface spécifiée continuera d'être annoncé dans les messages de routage envoyés via d'autres interfaces.

Par exemple, il est inutile que R1, R2 et R3 transfèrent des messages OSPF à partir de leurs interfaces de réseau local (LAN). La configuration identifie l'interface G0/0 de R1 comme étant passive.

Il est important de savoir qu'une contiguïté de voisinage ne peut pas être établie via une interface passive. La raison en est que les paquets LSP ne peuvent pas être envoyés ou reçus.

La commande show ip protocols est ensuite utilisée pour vérifier que l'interface Gigabit Ethernet était passive, comme illustré dans la Figure 2. Notez que l'interface G0/0 figure désormais dans la section Passive Interface(s). Le réseau 172.16.1.0 apparaît toujours sous Routing for Networks, ce qui signifie qu'il est encore inclus dans les mises à jour de routage OSPFv2 qui sont envoyées à R2 et R3.

Remarque : OSPFv2 et OSPFv3 prennent tous deux en charge la commande passive-interface.

Utilisez le contrôleur de syntaxe de la Figure 3 pour configurer l'interface LAN G0/0 en tant qu'interface passive sur R2.

Toutes les interfaces peuvent également être rendues passives à l'aide de la commande passive-interface default. Les interfaces qui ne doivent pas être passives peuvent être réactivées à l'aide de la commande no passive-interface.

Continuez à utiliser le contrôleur de syntaxe de la Figure 3 pour configurer l'interface LAN G0/0 en tant qu'interface passive sur R3.

Remarque : lors de l'exécution du contrôleur de syntaxe, notez les messages d'état OSPFv2 lorsque les interfaces sont toutes converties en interfaces passives et lorsque les deux interfaces séries sont converties en interfaces non passives.

La Figure 1 affiche les commandes utilisées pour configurer une interface passive sur un routeur qui exécute le protocole OSPF. La figure commence en mode de configuration globale et la première commande est router OSPF 10, où cette valeur désigne un ID de processus arbitraire. Cette commande active le protocole de routage OSPF. La commande suivante est passive-interface gigabit ethernet 0/0. Cette commande empêche la transmission des messages de routage via l'interface du routeur. La Figure 2 présente le résultat de la commande show ip protocols. La ligne répertorie les interfaces passives, qui sont mises en surbrillance. Dans ce cas, l'interface gigabit ethernet 0/0 est répertoriée comme étant passive.
