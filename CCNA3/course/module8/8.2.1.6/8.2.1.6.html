Utilisation d'une interface de bouclage comme ID de routeur

Un ID de routeur peut également être affecté au moyen d'une interface de bouclage.

L'adresse IPv4 de l'interface de bouclage doit être configurée avec un masque de sous-réseau 32 bits (255.255.255.255). Cette méthode permet de créer une route d'hôte de façon efficace. Une route d'hôte 32 bits n'est pas annoncée comme route aux autres routeurs OSPF.

L'exemple de la figure indique comment configurer une interface de bouclage avec une route d'hôte sur R1. R1 utilise la route d'hôte comme ID de routeur, à supposer qu'aucun ID de routeur n'ait été explicitement configuré ou appris précédemment.

Remarque : la commande router-id est la méthode recommandée. Toutefois, certaines versions antérieures de Cisco IOS ne reconnaissent pas la commande router-id. Dans ce cas, le meilleur moyen de définir l'ID de routeur consiste à utiliser une interface de bouclage.

La Figure 1 affiche les commandes utilisées pour configurer une interface de bouclage sur un routeur pour servir d'ID de routeur. La figure commence en mode de configuration globale. La première commande est interface loopback zero, qui passe en mode de configuration d'interface. La commande suivante est ip address 1.1.1.1 255.255.255.255. Cette commande définit l'adresse IP. Il s'agit désormais de l'ID de routeur.
