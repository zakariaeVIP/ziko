La Figure 1 montre les commandes utilisées pour attribuer une interface à une zone OSPF. La figure commence en mode de configuration globale. La première commande est router OSPF 10, où cette valeur désigne un ID de processus arbitraire. Cette commande active le protocole de routage OSPF.  La commande suivante permet d'identifier les interfaces qui participeront au routage OSPF. Elle se base sur l'adresse IP et le masque de sous-réseau.  Dans la figure, la commande est network 172.16.1.0 0.0.0.255 area 0.  Cette commande active OSPF au niveau de l'interface qui se trouve sur le réseau spécifié.  La commande suivante est network 172.16.1.0 0.0.0.3 area 0.  La dernière commande est network 192.168.10.4 0.0.0.255 area 0.  La Figure 2 montre les commandes utilisées pour attribuer des interfaces à une zone OSPF avec quatre zéros.  La figure commence en mode de configuration globale et la première commande active OSPF comme protocole de routage. Cette commande est router OSPF 10, où cette valeur désigne un ID de processus arbitraire.  La commande suivante permet d'identifier les interfaces qui participeront au routage OSPF. Elle se base sur l'adresse IP et un masque générique à quatre zéros.  Dans la figure, la commande est network 172.16.1.1 0.0.0.0 area 0.  La commande suivante est network 172.16.3.1 0.0.0.0 area 0.  La dernière commande est network 192.168.10.5 0.0.0.0 area 0.  La Figure 3 illustre un élément multimédia interactif qui permet à l'étudiant d'annoncer les réseaux OSPF.
