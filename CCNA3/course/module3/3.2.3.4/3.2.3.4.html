Les types de liaisons

Le type de lien catégorise chaque port participant à RSTP en utilisant le mode bidirectionnel du port. Suivant l'élément connecté à chaque port, deux types de lien peuvent être identifiés :

  • Point à point : un port fonctionnant en mode full duplex connecte généralement un commutateur à un commutateur et permet une transition rapide vers un état de réacheminement.

  • Partagé : un port fonctionnant en mode semi-duplex connecte un commutateur à un hub qui relie plusieurs appareils.

Dans cette figure, cliquez sur chaque lien pour en savoir plus sur son type.

Le type de lien détermine si le port peut passer immédiatement à l'état de réacheminement, sous réserve que certaines conditions soient remplies. Ces conditions ne sont pas les mêmes pour les ports d’extrémité et les autres types de ports. Les ports hors périphérie sont catégorisés en deux types de liaison : point à point et partagée. Le type de liaison est déterminé automatiquement, mais peut être remplacé par une configuration de port explicite à l'aide de la commande spanning-tree link-type { point-to-point | shared }. En matière de types de liaisons, les caractéristiques des rôles de port sont les suivantes :

  • Les connexions de port de périphérie et point à point sont des candidats pour une transition rapide à un état de réacheminement. Cependant, avant de prendre en compte le paramètre de type de lien, RSTP doit déterminer le rôle du port.

  • Les ports racine n'utilisent pas le paramètre de type de lien. Les ports racines sont capables de passer rapidement à l'état de réacheminement dès que le port est synchronisé (il reçoit un BPDU du pont racine).

  • Les ports alternatifs et de sauvegarde n'utilisent pas le paramètre de type de lien dans la majorité des cas.

  • Le paramètre de type de lien est principalement utilisé par les ports désignés. Le port désigné n'effectue une transition rapide à l'état de réacheminement que si le paramètre de type de lien est défini sur point à point.

La figure présente trois commutateurs qui sont tous interconnectés. Le commutateur 1 est identifié comme pont racine et l'ensemble de ses ports sont des ports désignés et de réacheminement. Les ports des commutateurs deux et trois sont identifiés en tant que port racine, désigné, alternatif et de périphérie, selon le cas.
