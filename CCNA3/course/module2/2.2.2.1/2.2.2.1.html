Erreurs au niveau des adresses IP et des masques de sous-réseau

Les VLAN correspondent à des sous-réseaux uniques sur le réseau. Pour que le routage inter-VLAN fonctionne, un routeur doit être connecté à tous les VLAN, par des interfaces physiques distinctes ou des sous-interfaces. Une adresse IP correspondant au sous-réseau pour lequel elle est connectée doit être affectée à chaque interface ou sous-interface. Ceci permet aux périphériques du VLAN de communiquer avec l'interface de routeur et d'activer le routage de trafic vers d'autres VLAN connectés au routeur.

Voici quelques erreurs courantes d'adressage IP :

  • Comme l'illustre la figure 1, le routeur R1 a été configuré avec une adresse IPv4 incorrecte sur l'interface G0/0. Ceci empêche PC1 de communiquer avec le routeur R1 sur le VLAN 10. Pour corriger ce problème, affectez l'adresse IPv4 correcte à l'interface G0/0 du routeur R1 à l'aide de la commande ip address 172.17.10.1 255.255.255.0. Une fois l'adresse IP correcte affectée à l'interface du routeur, le PC1 peut utiliser cette dernière comme passerelle par défaut pour accéder à d'autres VLAN.

  • Dans la figure 2, PC1 a été configuré avec une adresse IPv4 incorrecte pour le sous-réseau associé à VLAN 10. Ceci empêche PC1 de communiquer avec le routeur R1 sur le VLAN 10. Pour corriger ce problème, affectez l'adresse IPv4 correcte à PC1. Selon le type de PC utilisé, les détails de configuration peuvent être différents.

Dans la Figure 3, le PC1 a été configuré avec un masque de sous-réseau incorrect. Selon le masque de sous-réseau configuré pour PC1, PC1 se trouve sur le réseau 172.17.0.0. Il en résulte que PC1 calcule que PC3, dont l'adresse IPv4 est 172.17.30.23, se trouve dans le même sous-réseau que lui. Par conséquent, le PC1 ne transmet pas le trafic destiné au PC3 à l'interface G0/0 du routeur R1. Ce trafic n'atteint donc jamais le PC3. Pour corriger ce problème, changez le masque de sous-réseau sur PC1 en 255.255.255.0. Selon le type de PC utilisé, les détails de configuration peuvent être différents.

La figure 1 montre que la sous-interface d'un routeur connecté à une topologie de routage inter-VLAN existante a été configurée avec une adresse IP incorrecte. Cela empêche les hôtes de ce VLAN d'être correctement acheminés vers d'autres VLAN. La figure 2 présente un hôte configuré avec une adresse IP incorrecte. Cela empêche l'hôte de communiquer avec le routeur d'un VLAN différent. La figure 3 présente un hôte configuré avec l'adresse IP « 172 dot 17 dot 10 dot 21 slash 16 » sur un VLAN et un hôte situé sur un VLAN différent configuré avec l'adresse IP « 172 dot 17 dot 30 dot 23 slash 24 ».
