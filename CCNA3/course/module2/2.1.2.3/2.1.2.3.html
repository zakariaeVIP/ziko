Étape 2 : Configuration du nom de domaine et du mot de passe VTP

Le nom de domaine est configuré à l'aide de la commande de configuration globale vtp domain domain-name . Dans la figure 1, le nom de domaine configuré est CCNA sur S1. S1 envoie ensuite une annonce VTP à S2 et S3. Si S2 et S3 possèdent la configuration par défaut avec le nom de domaine NULL, les deux commutateurs accepteront CNNA comme nouveau nom de domaine VTP. Un client VTP doit avoir le même nom de domaine que le serveur pour accepter les annonces VTP.

Pour des raisons de sécurité, un mot de passe doit être configuré avec la commande vtp password password . Dans la figure 2, le mot de passe cisco12345 est défini pour le domaine VTP. Tous les commutateurs du domaine VTP doivent utiliser le même mot de passe de domaine VTP.

Vérifiez le mot de passe VTP au moyen de la commande show vtp password, comme illustré à la figure 2.

La figure 1 montre comment configurer le nom de domaine. La figure 2 montre comment configurer et vérifier le mot de passe du nom de domaine.
