Résolution des problèmes de protocole VTP

Les problèmes courants liés à VTP sont les suivants :

Versions VTP incompatibles

  • Les versions de VTP sont incompatibles les unes avec les autres.

  • Vérifiez que tous les commutateurs peuvent prendre en charge la version de VTP requise.

Problèmes liés aux mots de passe VTP

  • Si l'authentification VTP est activée, le même mot de passe doit être configuré sur les commutateurs pour qu'ils puissent participer à VTP.

  • Veillez à ce que le mot de passe soit configuré manuellement sur tous les commutateurs du domaine VTP.

Nom de domaine VTP incorrect

  • Un domaine VTP mal configuré affecte la synchronisation des VLAN entre les commutateurs et si un commutateur reçoit une mauvaise annonce VTP, il supprime le message.

  • Pour éviter toute configuration incorrecte du nom de domaine VTP, définissez-le sur un seul commutateur de serveur VTP.

  • Tous les autres commutateurs du même domaine VTP accepteront et configureront automatiquement leur nom de domaine VTP en recevant la première annonce VTP de type résumé.

Définition de tous les commutateurs en mode client VTP

  • Si tous les commutateurs du domaine VTP sont configurés en mode client, vous ne pouvez pas créer, supprimer et gérer les VLAN.

  • Pour éviter de perdre toutes les configurations de VLAN d'un domaine VTP, configurez deux commutateurs comme serveurs VTP.

Numéro de révision de la configuration incorrect

  • En cas d'ajout au domaine d'un commutateur portant le même nom de domaine VTP mais un numéro de configuration plus élevé, des VLAN non valides risquent d'être propagés et/ou des VLAN valides peuvent être supprimés.

  • La solution consiste à réinitialiser chaque commutateur à une configuration antérieure puis à reconfigurer les VLAN corrects.

  • Avant d'ajouter un commutateur à un réseau compatible VTP, réinitialisez à 0 le numéro de révision sur le commutateur en l'affectant à un autre faux domaine VTP puis en le réaffectant au nom de domaine VTP correct.
