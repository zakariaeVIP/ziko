Chapitre 7 : réglage et dépannage du protocole EIGRP

Le protocole EIGRP est l'un des protocoles de routage les plus couramment utilisés dans les réseaux de grandes entreprises. Un ingénieur réseau impliqué dans l'implémentation et la maintenance de grands réseaux d'entreprise routés utilisant le protocole EIGRP doit absolument savoir comment modifier les fonctionnalités EIGRP et résoudre les problèmes rencontrés.

La récapitulation permet de réduire le nombre d'entrées dans les mises à jour de routage, ainsi que dans les tables de routage locales. Elle réduit également la consommation de bande passante par les mises à jour de routage et accélère les recherches dans les tables de routage. La récapitulation automatique du protocole EIGRP pour IPv4 est désactivée par défaut à partir des versions Cisco IOS 15.0(1)M et 12.2(33). Auparavant, la récapitulation automatique était activée par défaut. Pour activer la récapitulation automatique du protocole EIGRP, utilisez la commande auto-summary en mode de configuration du routeur. Utilisez la commande show ip protocols pour vérifier l'état de la récapitulation automatique. Examinez la table de routage pour vérifier le bon fonctionnement de la récapitulation automatique.

Le protocole EIGRP inclut automatiquement des routes récapitulatives vers Null0 pour éviter les boucles de routage présentes dans le récapitulatif, mais absentes de la table de routage. L'interface Null0 est une interface IOS virtuelle qui ne conduit nulle part, également appelée « le grand vide ». Les paquets qui correspondent à une route avec une interface de sortie Null0 sont mis à l'écart.

Une méthode de propagation d'une route par défaut dans le domaine de routage EIGRP consiste à utiliser la commande redistribute static. Cette commande demande au protocole EIGRP d'inclure la route statique dans ses mises à jour EIGRP vers d'autres routeurs. La commande show ip protocols vérifie que les routes statiques du domaine de routage EIGRP sont bien redistribuées.

Utilisez la commande ip bandwidth-percent eigrp as-number percent en mode de configuration d'interface pour configurer le pourcentage de bande passante pouvant être utilisé par le protocole EIGRP sur une interface.

Pour configurer le pourcentage de la bande passante pouvant être utilisé par le protocole EIGRP pour IPv6 sur une interface, utilisez la commande ipv6 bandwidth-percent eigrp en mode de configuration d'interface. Pour restaurer la valeur par défaut, utilisez la forme no de cette commande.

Les intervalles Hello et les temps d'attente peuvent être configurés pour chaque interface dans EIGRP et ne doivent pas forcément correspondre à ceux des autres routeurs EIGRP pour établir ou conserver des contiguïtés.

Pour IP dans le protocole EIGRP, le logiciel Cisco IOS applique l'équilibrage de la charge à l'aide de quatre chemins à coût égal par défaut, au maximum. À l'aide de la commande de mode de configuration du routeur maximum-paths, vous pouvez conserver jusqu'à 32 routes à coût égal dans la table de routage.

La commande show ip route permet de vérifier que le routeur a appris les routes EIGRP. La commande show ip protocols permet de vérifier que le protocole EIGRP affiche les valeurs actuellement configurées.
