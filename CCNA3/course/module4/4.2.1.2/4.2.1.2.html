Configuration des interfaces

La configuration d'un canal EtherChannel avec le protocole LACP se déroule en deux étapes :

Étape 1. Indiquez les interfaces qui composent le groupe EtherChannel à l'aide de la commande interface range interface en mode de configuration globale. Le mot-clé range vous permet de sélectionner plusieurs interfaces et de les configurer toutes ensemble. Il est recommandé de commencer par désactiver ces interfaces, de sorte qu'aucune configuration incomplète n'entraîne d'activité sur la liaison.

Étape 2. Créez l'interface de canal de port à l'aide de la commande channel-group identifier mode active en mode de configuration de plage d'interface. L'identificateur spécifie un numéro de groupe de canaux. Les mots-clés mode active l'identifient comme une configuration EtherChannel LACP.

Remarque : EtherChannel est désactivé par défaut.

Dans la Figure 1, FastEthernet0/1 et FastEthernet0/2 sont regroupés dans le canal de port de l'interface EtherChannel 1.

Pour modifier les paramètres de couche 2 sur l'interface de canal de port, activez le mode de configuration d'interface de canal de port en saisissant la commande interface port-channel, suivie de l'identifiant de l'interface. Dans l'exemple, l'EtherChannel est configuré en tant qu'interface trunk avec des VLAN autorisés spécifiés. Comme le montre la Figure 1, le canal de port est configuré en tant que trunk autorisant le trafic en provenance des VLAN 1, 2 et 20.

Utilisez le contrôleur de syntaxe de la Figure 2 pour configurer EtherChannel sur le commutateur S1.

La Figure 1 répertorie les commandes nécessaires pour configurer un canal EtherChannel. Les commandes commencent en mode de configuration globale. La première commande est interface range fast Ethernet 0/1-2. Elle permet d'appliquer les autres commandes saisies aux ports 1 et 2. La commande suivante, channel-group 1 mode active, crée l'interface de canal de port. La commande suivante, interface port-channel 1, permet de passer au mode de configuration pour le canal de port. La commande switchport mode trunk configure l'interface en tant qu'interface trunk. La dernière commande, switchport trunk allowed v lan 1,2,20, configure les VLAN autorisés sur la liaison.
