La commande network

Le mode de configuration du routeur EIGRP permet de configurer le protocole de routage EIGRP. La Figure 1 montre que les routeurs R1, R2 et R3 disposent tous de réseaux qui doivent intégrer un même domaine de routage EIGRP. Pour activer le routage EIGRP sur une interface, utilisez la commande de mode de configuration de routeur network ipv4-network-address . L'adresse ipv4-netword-address correspond à l'adresse réseau par classe pour chaque réseau connecté directement.

La commande network a la même fonction que dans tous les protocoles de routage IGP. La commande network dans le protocole EIGRP a la fonction suivante :

  • Elle permet à chaque interface du routeur correspondant à l'adresse réseau indiquée dans la commande de mode de configuration du routeur network d'envoyer et de recevoir des mises à jour EIGRP.

  • Le réseau des interfaces est inclus dans les mises à jour de routage EIGRP.

La Figure 2 présente les commandes network requises pour configurer EIGRP sur le routeur R1. Dans la figure, une seule instruction network par classe, network 172.16.0.0, est utilisée sur R1 pour inclure les interfaces dans les sous-réseaux 172.16.1.0/24 et 172.16.3.0/30. Notez que seule l'adresse réseau par classe est utilisée.

La Figure 3 montre la commande network servant à activer le protocole EIGRP sur les interfaces de R2 pour les sous-réseaux 172.16.1.0/24 et 172.16.2.0/24. Lorsque le protocole EIGRP est configuré sur l'interface S0/0/0 de R2, DUAL envoie un message de notification à la console pour indiquer qu'une contiguïté de voisinage a été établie avec un autre routeur EIGRP sur cette interface. Cette nouvelle contiguïté se produit automatiquement, car les routeurs R1 et R2 utilisent le même numéro de système autonome (à savoir, 1) ; par ailleurs, ils envoient désormais tous deux des mises à jour sur leurs interfaces dans le réseau 172.16.0.0.

DUAL génère automatiquement le message de notification, car la commande de mode de configuration de routeur eigrp log-neighbor-changes est activée par défaut. Plus précisément, cette commande permet de vérifier les contiguïtés de voisinage pendant la configuration du protocole EIGRP et d'afficher toutes les modifications dans les contiguïtés de voisinage EIGRP, notamment lors de l'ajout ou de la suppression d'une contiguïté EIGRP.
