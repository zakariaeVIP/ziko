Chapitre 8 : OSPF à zone unique

La version actuelle du protocole OSPF pour IPv4 est OSPFv2, présenté dans le document RFC 1247 et mis à jour dans le document RFC 2328 par John Moy. En 1999, OSPFv3 pour IPv6 a été publié dans le document RFC 2740.

OSPF est un protocole de routage à état de liens dont la distance administrative par défaut s'élève à 110. Dans la table de routage, le code O y fait référence.

Le protocole OSPFv2 est activé avec la commande router ospf process-id en mode de configuration globale. La valeur process-id est une valeur locale. Autrement dit, elle n'a pas besoin d'être identique sur les autres routeurs OSPFv2 pour établir la contigüité avec les voisins.

La commande network utilisée avec OSPFv2 a la même fonction qu'avec les protocoles de routage IGP. Seule la syntaxe diffère légèrement. La valeur wildcard-mask est l'inverse du marque de sous-réseau, et la valeur area-id doit être définie sur 0.

Par défaut, des paquets Hello OSPF sont envoyés toutes les 10 secondes sur les segments à accès multiple et point à point, et toutes les 30 secondes sur les segments NBMA (Frame Relay, X.25, ATM), et sont utilisés par OSPF pour établir des contiguïtés de voisinage. Par défaut, l'intervalle Dead est quatre fois plus long que l'intervalle Hello.

Pour que les routeurs deviennent contigus, leurs intervalles de paquets Hello et Dead, leurs types de réseau et leurs masques de sous-réseau doivent correspondre. Utilisez la commande show ip ospf neighbor sur R1 pour vérifier les contiguïtés OSPFv2.

OSPF sélectionne un routeur désigné (DR) pour servir de point de collecte et de distribution pour les paquets LSA envoyés et reçus dans le réseau à accès multiple. Un routeur désigné de secours est sélectionné pour remplacer le routeur désigné si ce dernier venait à défaillir. Tous les autres routeurs sont appelés DROTHER. Tous les routeurs envoient leur LSA au DR, qui les diffuse ensuite aux autres routeurs du réseau à accès multiple.

La commande show ip protocols permet de vérifier les informations de configuration OSPFv2 essentielles, notamment l'ID de processus OSPF, l'ID de routeur et les réseaux annoncés par le routeur.

OSPFv3 est activé sur une interface et n'est pas en mode de configuration du routeur. OSPFv3 requiert la configuration des adresses link-local. Le routage monodiffusion IPv6 doit être activé pour OSPFv3. Un ID de routeur de 32 bits est requis avant qu'une interface puisse être activée pour OSPFv3. Des commandes de vérifications analogues à celles d'OSPFv2 sont utilisées pour OSPFv3.
